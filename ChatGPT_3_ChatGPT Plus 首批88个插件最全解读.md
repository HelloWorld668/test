如何创建与插件 API 的请求：

创建图表时的重要规则：
- 更喜欢使用 graph TB 类型的图表。
- 尽可能避免使用线性图表，图表应具有层次结构，并在适用时具有多个分支。
- 在图表中不要使用 ampersand (&) 符号，它会破坏图表。请改用单词 "and"。例如，使用 "User and Admin" 而不是 "User & Admin"。
- 在节点标识符、节点标签和边标签中，不要使用圆括号 ()，它会破坏图表。请使用逗号代替。例如，使用 "User, Admin" 而不是 "User (Admin)"。
- 不要为边添加空标签 ""，而是不要对边进行标记。例如 U["User"] --> A["Admin"]。
- 如果标签与目标节点相同，则不要添加标签。

使用图形图表时的规则：
- 使用简短的节点标识符，例如 U 表示 User 或 FS 表示 File System。
- 节点标签始终使用双引号，例如 U["User"]。
- 边标签始终使用双引号，例如 U["User"] -- "User enters email" --> V["Verification"]。
- 缩进非常重要，请始终按照以下示例进行缩进。

使用带有子图的图形图表时的规则：
不要在子图内部引用子图根节点。
